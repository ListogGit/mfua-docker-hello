# Развертывание PostgreSQL через Docker

Данное руководство описывает процесс быстрого запуска популярной реляционной системы управления базами данных PostgreSQL. Мы будем использовать оптимизированную и легковесную версию образа на базе Alpine Linux.

> **Важно:** Никогда в разработке не используйте русские имена файлов и каталогов! 
> Никогда не используйте пробелы и спецсимволы в именах файлов и каталогов!

## 1. Предварительная проверка
Убедитесь, что служба Docker установлена в вашей системе (Debian Linux) и функционирует корректно. Проверить это можно командой:

    docker --version

## 2. Инициализация контейнера
Для загрузки образа и запуска базы данных выполните следующую команду:

    docker run -d \
      --name my-postgres \
      -p 5432:5432 \
      -e POSTGRES_PASSWORD=mysecretpassword \
      postgres:alpine

**Расшифровка аргументов запуска:**
* `-d` — отсоединяет процесс от консоли (фоновый режим работы).
* `--name my-postgres` — присваивает контейнеру понятное имя для удобного управления.
* `-p 5432:5432` — пробрасывает порт хоста (5432) на стандартный порт PostgreSQL внутри контейнера (5432).
* `-e POSTGRES_PASSWORD=mysecretpassword` — задает обязательную переменную окружения: пароль для дефолтного суперпользователя `postgres`.
* `postgres:alpine` — указывает Docker использовать официальный образ PostgreSQL на базе минималистичной ОС Alpine (занимает меньше места и быстрее скачивается).

## 3. Мониторинг состояния
После выполнения команды убедитесь, что контейнер с базой данных успешно стартовал:

    docker ps -a

![alt text](./img1.png)

В выведенной таблице должен отображаться контейнер `my-postgres` со статусом `Up`. 

## 4. Подключение к базе данных через psql
Чтобы взаимодействовать с запущенной базой данных напрямую из терминала, необходимо подключиться к ней с помощью встроенной консольной утилиты `psql`:

    docker exec -it my-postgres psql -U postgres

![alt text](./img2.png)

**Что делает эта команда:**
* `docker exec -it` — запускает команду внутри уже работающего контейнера в интерактивном режиме.
* `my-postgres` — обращение к нашему запущенному контейнеру по имени.
* `psql -U postgres` — запуск клиента PostgreSQL от имени пользователя `postgres`. *(Пароль вводить не потребуется, так как команда выполняется локально внутри самого контейнера).*

В результате терминал изменится на приглашение вида `postgres=#`, где можно выполнять SQL-запросы. Для выхода обратно в терминал Debian введите `\q` и нажмите Enter.

## 5. Базовые команды управления
Для дальнейшего администрирования контейнера пригодятся эти команды:

* Остановка базы данных:
    docker stop my-postgres

* Повторный запуск:
    docker start my-postgres

* Полное удаление контейнера (Внимание: так как в этом примере не настроен Volume, все таблицы и данные БД будут потеряны при удалении контейнера!):
    docker rm -f my-postgres
